Software Architecture

The simulation environment is powered by three distinct software packages, each tailored to a specific role within the training exercise.

Server Control

  • Manages core simulation state
  • Shows client connection status
  • Provides ASR/PAR monitoring views
  • Includes server-level controls (e.g., scenarios, weather)

Officer Station

  • Full ASR and PAR visualization
  • Complete aircraft control system
  • Issues commands to aircraft
  • Receives real-time simulation data

Student Station

  • ASR and PAR visualization
  • Read-only access to aircraft data
  • No direct command input
  • Ideal for observation and learning

Hardware Configuration

The system is designed around a central server managing the simulation for multiple client stations.

  • 1x Server Computer: Runs the core simulation logic, manages all aircraft, and serves data to all clients.
  • 1x Network Switch: Provides a dedicated, high-speed local network for all simulator computers.
  • 7x Client Computers: Connect to the server to display radar views and interact with the simulation.

Client Allocation:

  • 5x Officer/Pilot Stations: These stations are for primary trainees, providing a full-featured radar interface.
  • 2x Student Stations: These stations can be used for observation or secondary tasks, potentially with a more limited set of controls.

Network & Cable Management

A clean and organized setup is crucial for maintenance and reliability.

  • Network: All computers will be connected via a dedicated, high-speed Local Area Network (LAN) using the central network switch to ensure low latency.
  • Cabling: All power and network cables will be managed using under-desk cable trays and raceways to prevent clutter and tripping hazards.
  • Peripherals: Each station will be equipped with a monitor, keyboard, mouse, and headset, with cables neatly routed.
