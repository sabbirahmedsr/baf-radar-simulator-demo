Radar Simulator

A real-world, practical simulator designed to train student controllers in a highly realistic environment. The system uses a central server to run the simulation, featuring an accurate map of the Bangladesh skies and professional-grade radar displays (ASR & PAR). Aircraft behave according to real-world physics, and an integrated voice system allows trainees to communicate directly with pseudo-pilots for a fully immersive training experience.

1.0 The Three Applications: Features & Functionality

The simulation environment is powered by three distinct software packages, each providing a specific set of features for the training exercise.

Instructor Application (Server)

Core Purpose:

This application is the central brain and core engine of the simulator. It runs the entire simulation, calculates all physics, and acts as the central server to synchronize every client.

Key Interfaces:
  • Airport Surveillance Radar (ASR) View
  • Precision Approach Radar (PAR) View
  • Geo-Accurate Bangladesh Map
  • Push-to-Talk (PTT) Master Voice Interface
  • Master Command & Control Panel
  • Session Lifecycle Control Panel
  • Exercise & Scenario Management System
  • Live Aircraft Creation & Control System
  • Network & Session Status Monitor
  • Advanced Environmental Control Panel
  • System & Aircraft Malfunction Injector
  • Simulation Time & Speed Control Panel
  • Map Measurement & Analysis Tools
Detailed Features:
  • Load and run exercises from a predefined list.
  • Manage the simulation session (start, pause, restart, end).
  • Inject real-time events like emergencies or weather shifts.
  • Take direct "pilot" control of any aircraft for hands-on intervention.
  • Override any voice channel to communicate directly with clients.
  • Control simulation speed with pause and fast-forward options.
  • Create new aircraft dynamically during a running session.
  • Modify any aircraft's flight plan or parameters in real-time.
  • Trigger specific malfunctions (e.g., engine failure).
  • Set precise environmental conditions like wind layers and visibility.
  • Use on-map tools for range and bearing measurements.

Officer Application (Client as Pilot)

Core Purpose:

This application puts the user in the role of a pseudo-pilot. They listen to verbal commands from the student controller, then manually input those instructions—like heading, altitude, and speed—into the aircraft's command panel. The user also provides manual voice responses back to the student using the PTT system, completing the communication loop.

Key Interfaces:
  • Airport Surveillance Radar (ASR) View
  • Precision Approach Radar (PAR) View
  • Geo-Accurate Bangladesh Map
  • Command & Control Panel
  • System & Aircraft Malfunction Injector
  • Map Measurement & Analysis Tools
  • PTT Voice & Channel Monitor
Detailed Features:
  • Listen for verbal commands from student controllers.
  • Provide voice readbacks to controllers via PTT.
  • Manually execute ATC commands (heading, altitude, speed).
  • Display live aircraft data received from the server.
  • Switch between assigned PTT voice channels.
  • Activate aircraft malfunctions on instructor command.
  • View the flight plan for the assigned aircraft.

Student Application (Client as Controller)

Core Purpose:

This application places the student in the role of an Air Traffic Controller. They observe aircraft on high-fidelity ASR and PAR radar displays, analyze traffic situations, and issue verbal commands to pilots via the PTT voice system to manage the airspace.

Key Interfaces:
  • Airport Surveillance Radar (ASR) Scope
  • Precision Approach Radar (PAR) Scope
  • Geo-Accurate Bangladesh Map
  • Push-to-Talk (PTT) Voice Interface
  • On-Screen Radar Measurement Tools
Detailed Features:
  • Ensure airspace safety and guide aircraft for safe landings.
  • Issue verbal commands to pilots via the PTT system.
  • Monitor aircraft on ASR, PAR, and map displays.
  • Maintain vertical and horizontal separation between aircraft.
  • Analyze aircraft trajectories using history trails and vector lines.
  • Use on-screen tools for range, bearing, and separation checks.
  • Identify and manage potential conflicts between aircraft.
  • Switch between ASR for terminal area and PAR for final approach.

2.0 Core System Modules

Simulation Engine

The server-side "heart" of the simulator. Updates aircraft state based on physics and commands. (e.g., physics.js)

Networking Module

Handles all data synchronization between server and clients using WebSockets. (e.g., network.js)

Voice Module

Manages the internal PTT voice chat using WebRTC, simulating radio channels with instructor override. (e.g., voice.js)

Radar Display Module

Renders the ASR/PAR scopes, including aircraft blips, data blocks, and history trails on HTML5 Canvas. (e.g., display.js)

Command & Control Module

The UI for Instructors and Officers to control the simulation and their assigned aircraft. (e.g., controls.js)

Exercise & Scenario Module

Manages loading, running, and saving training exercises from JSON/JS files. (e.g., scenario.js)

3.0 Detailed Feature Breakdown

Radar Display & Aircraft Features

Interactive Map Node Label

Draggable UI nodes for each aircraft showing callsign, altitude, speed, and squawk code. Can be toggled on/off.

History & Velocity Vectors

Displays aircraft history trails and projected velocity vectors to help controllers predict future positions. Can be toggled on/off.

On-Screen Measurement Tools

Provides digital tools for measuring range and bearing between any two points or aircraft directly on the radar scope.

Map & Navigation Features

Real-World Coordinate System

The map is built on a real-world projection, converting actual latitude and longitude into a geo-accurate grid for precise positioning.

Dynamic Map Layers

Toggle visibility for different map types, including domestic/international routes, restricted areas, and major reporting points.

Arrival & Approach Visualization

Select and display standard instrument arrival (STAR) and approach procedures directly on the map for detailed analysis.
